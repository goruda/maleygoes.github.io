"Euler's Method" (and why we really shouldn't call it that)

When do we care about this?

This is a beginner's level idea for how we can numerically approximate solutions to IVP's. When we can't solve ODEs by hand (most of them) we must resort to something!

How do we do it?

We start with the initial condition, \( (x_0,y_0) \) and then moving forward through the slope field given by \( y' = f(x,y) \). We take tiny steps and hope that we land on points close to the solution.

It is a bit like trying to draw a curve with tiny straight lines, like we did when we were kids. Because that's almost exactly what we're doing.

What do we need to remember?

You need to remember that we call the function \(y=f(x)\) a solution to the DE \( y' = f(x,y) \) and an integral curve is the geometric stand-in for a solution.

You should also refresh yourself on what the tangent line is if you forgot...

Why did you say we shouldn't call it "Euler's Method"?

Well, everyone does. But Euler would have hated it. He came up with it as a pseudo-method in order to solve a larger problem. Unto itself, he never intended it to be thought of as a legit method to solve DEs. Because it's really not. BUT it is a great illustrative idea.
